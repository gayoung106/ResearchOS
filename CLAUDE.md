# Python 연구분석 자동화 프로젝트 지침

## 1. 프로젝트 목적

이 프로젝트는 사회과학, 행정학, 정책학, 조직연구 및 법정책 연구에서 사용하는 재사용 가능한 Python 기반 연구분석 프레임워크다.

매 프로젝트마다 다음 항목이 달라질 수 있다.

- 연구주제
- 연구질문
- 가설
- 자료구조
- 설문지와 코드북
- 변수명
- 변수의 측정수준
- 분석방법
- 표본설계
- 가중치와 군집구조
- 결과표 형식

따라서 이전 연구의 변수 정의나 분석방법을 현재 연구에 기계적으로 적용하지 않는다.

---

## 2. 언어 및 출력 원칙

1. 모든 설명, 보고서, 표 제목, 그림 제목, 코드 주석은 한국어로 작성한다.
2. Python 함수명, 패키지명, 변수 원명은 필요한 경우 영어로 유지한다.
3. 논문에 직접 사용할 수 있는 표와 문장을 생성한다.
4. 모든 결과 파일은 `result/` 아래에 저장한다.
5. 원자료는 절대 수정하거나 덮어쓰지 않는다.
6. 추측한 정보는 사실처럼 작성하지 않는다.
7. 확인하지 못한 서지정보, DOI, 저자명, 연도, 페이지를 생성하지 않는다.

---

## 3. 프로젝트 디렉터리 규칙

- `rawdata/`: 실제 원자료
- `questionnaire/`: 설문지, 코드북, 레이아웃, 변수설명서
- `config/`: 연구설계 및 분석설정
- `src/`: Python 코드
- `result/`: 분석 결과
- `result/logs/`: 실행 기록과 오류 로그
- `manuscript/`: 연구방법 및 연구결과 초안
- `tests/`: 단위 및 통합 테스트

원자료와 결과 파일은 Git에 포함하지 않는다.

## 4. 연구 시작 절차

새로운 연구에서는 바로 분석 코드를 작성하지 않는다.

다음 순서로 진행한다.

1. 연구주제 확인
2. 연구질문 확인
3. 가설 확인
4. 분석단위 확인
5. 자료구조 확인
6. 설문지 및 코드북 확인
7. 변수의 역할 확인
8. 변수의 측정수준 확인
9. 분석방법 후보 제안
10. 분석 전제조건 및 한계 검토
11. 연구설계 YAML 작성
12. 연구자의 승인 후 코드 생성

필요한 정보가 부족하더라도 임의로 단정하지 않는다.
확정된 정보와 잠정적 판단을 구분해서 기록한다.

## 6. 파일명 및 코드 작성 규칙

1. 실행 순서는 두 자리 번호를 사용한다.
2. 파일명은 한글 설명을 포함할 수 있다.
3. 모든 실행 파일에는 `main()` 함수를 둔다.
4. 모든 실행 파일에는 다음 블록을 둔다.

```python
if __name__ == "__main__":
    main()
```

5. 절대경로를 하드코딩하지 않는다.
6. 경로는 `src/common/paths.py`에서 가져온다.
7. 공통 기능은 `src/common/`에 작성한다.
8. 통계 분석 함수는 `src/statistics/`에 작성한다.
9. 표 생성은 `src/reporting/`에 작성한다.
10. 심사 및 검토 로직은 `src/review/`에 작성한다.

## 8. 변수 측정수준 판별 규칙

변수는 다음 유형 중 하나로 분류한다.

- 이분형
- 명목형
- 순서형
- 연속형
- 횟수형
- 비율형
- 날짜형
- 문자열
- 다중응답
- 척도 구성 문항
- 식별 불가

판별 우선순위는 다음과 같다.

1. 설문지 문항 원문
2. 코드북
3. 변수 라벨
4. 값 라벨
5. 관측값과 빈도
6. 변수명과 인접 문항

다음과 같이 단정하지 않는다.

- 값이 0과 1이라는 이유만으로 이분형이라고 단정
- 값이 1과 2라는 이유만으로 이분형이라고 단정
- 값이 1부터 5라는 이유만으로 리커트 척도라고 단정
- 숫자형이라는 이유만으로 연속형이라고 단정

판단 근거가 부족하면 `검토 필요`로 분류한다.

가능한 전처리 항목은 다음과 같다.

- 이분형 0/1 재코딩
- 범주 통합
- 기준범주 지정
- 역문항 처리
- 척도 평균 또는 합산
- 날짜 파싱
- 파생변수 생성
- 중심화
- 표준화
- 극단값 처리
- 완전사례 표본 생성
- 다중응답 정리

모든 변환은 다음 정보를 남긴다.

- 변수명
- 원래 값
- 변환된 값
- 변환식
- 처리 근거
- 처리 전 빈도

- 결측치 변화

원자료는 절대 변경하지 않는다.

## 10. 분석방법 선택 규칙

분석방법은 종속변수 측정수준, 자료구조, 표본설계 및 연구질문에 따라 결정한다.

- 연속형 종속변수: OLS 또는 일반화선형모형
- 이분형 종속변수: 로지스틱 또는 프로빗
- 순서형 종속변수: 순서형 로짓 또는 프로빗
- 명목형 다범주 종속변수: 다항로짓
- 횟수형 종속변수: 포아송 또는 음이항
- 패널자료: 고정효과, 임의효과, GEE
- 군집자료: 다층모형 또는 군집강건표준오차
- 잠재변수: CFA 또는 SEM
- 매개효과: 부트스트랩 간접효과
- 조절효과: 상호작용 및 예측값
- 집단비교: 측정동일성 및 계수차이 검정
- 인과추론: 식별전략이 확보된 경우만 적용

사용자가 요청한 분석이 자료와 맞지 않으면 문제점과 대안을 먼저 제시한다.

## 15. 시각화 규칙

1. 연구질문과 직접 관련된 그림만 생성한다.
2. 축과 범례는 한국어로 작성한다.
3. 신뢰구간을 가능한 경우 표시한다.
4. 상호작용은 예측값 또는 평균한계효과로 제시한다.
5. 3차원 그래프를 사용하지 않는다.
6. 색상만으로 집단을 구분하지 않는다.
7. 그림 생성용 데이터도 CSV로 저장한다.
8. PNG와 SVG를 함께 저장한다.

## 16. 논문 문장 생성 규칙

연구방법 및 연구결과 초안은 실제 분석 출력값만 사용한다.

다음 표현을 구분한다.

- 통계적으로 유의함
- 통계적으로 유의하지 않음
- 효과크기가 큼 또는 작음
- 가설 지지
- 부분 지지
- 지지되지 않음
- 자료만으로 판단 불가

p값이 0.05보다 크다고 효과가 없다고 단정하지 않는다.
통계적으로 유의하다고 이론적으로 중요하다고 단정하지 않는다.
횡단면 관찰자료에서 인과관계를 단정하지 않는다.

## 19. 작업 완료 조건

다음 조건을 충족해야 분석 완료로 판단한다.

- 원자료가 변경되지 않음
- 코드가 순서대로 실행됨
- 분석 데이터가 재현 가능하게 생성됨
- 전처리 로그가 존재함
- 표본 흐름이 기록됨
- 변수 정의가 문서화됨
- 수렴 여부와 경고가 기록됨
- 진단 및 강건성 검토가 완료됨
- 논문용 표와 그림이 생성됨
- 연구방법 및 연구결과 초안이 작성됨
- 다중역할 검토가 완료됨
- 표와 본문의 수치가 일치함
- 한계와 주장 범위가 명시됨

분석 결과가 기대와 다르더라도 결과를 변경하거나 선택적으로 누락하지 않는다.
